# Daily Status - 2026-03-15

## Accomplishments
- Continued work on [OSAC ClusterOrder AAP Direct Ready](https://github.com/osac-project/osac-operator/pull/110) — rebased on upstream main and committed fixes for block-deletion-on-failure behavior and UTC timestamp handling
- Implemented config version tracking and duplicate trigger guard for ClusterOrder and HostPool resources, ensuring spec changes are detected and re-provisioned while preventing duplicate triggers from stale cache during back-to-back reconciles
- Added comprehensive unit tests covering provision trigger logic, config version handling, and idempotency guarantees
- Deployed to vmaas-dev and performed end-to-end testing — verified ClusterOrder and HostPool provisioning, job polling, and config version tracking all work correctly
- Discovered a critical bug during E2E testing: failed provision jobs cause an infinite retry loop because the reconciled config version is never set on failure, causing perpetual re-triggering — this bug also affects ComputeInstance but had gone unnoticed
- Drafted a proposed fix and multiple approaches for the infinite retry bug to discuss with Adrien (planned as a separate PR affecting all three resource types)
- Added integration tests for ClusterOrder (10 tests) and HostPool (8 tests) covering the full provision lifecycle, failure handling, stale cache guard, deprovisioning, and config version behavior — including pending tests that document the known infinite retry bug
- Pushed all changes to [osac-operator#110](https://github.com/osac-project/osac-operator/pull/110)
