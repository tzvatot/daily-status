# Daily Status - 2026-04-14

## Accomplishments
- **[MGMT-23659](https://issues.redhat.com/browse/MGMT-23659) — NetworkClass ID should use UUID instead of FQN:** Introduced a new fully-qualified name (FQN) field for NetworkClass resources so that the server-generated UUID is used as the canonical ID while preserving a human-readable identifier. Updated the NetworkClass creation and update flows to enforce FQN requirements, and adapted the template publishing logic to use FQN-based lookups. All unit tests pass across both [fulfillment-service#395](https://github.com/osac-project/fulfillment-service/pull/395) and [osac-aap#242](https://github.com/osac-project/osac-aap/pull/242) (both in draft).

## Risks & Challenges
None

## Key Effort
Switching NetworkClass identifiers from fully-qualified names to server-generated UUIDs while preserving FQN as a separate field for lookups and publishing.
