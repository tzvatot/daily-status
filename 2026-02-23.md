# Daily Status - 2026-02-23

## Accomplishments

- **[MGMT-23031](https://issues.redhat.com/browse/MGMT-23031) - Changed default provisioning provider from EDA to AAP** in osac-operator (`cmd/main.go` and config sample updated). All tests pass, linter clean.
- **[MGMT-23103](https://issues.redhat.com/browse/MGMT-23103) - Fixed re-provision logic in operator** — added `needsProvisionJob()` receiver to handle spec changes (e.g. runStrategy) triggering new provision jobs. Deployed to vmaas-dev.
- **[MGMT-23103](https://issues.redhat.com/browse/MGMT-23103) - Fixed E2E test** to check `spec.runStrategy` instead of deprecated `spec.running`.
- **[MGMT-23103](https://issues.redhat.com/browse/MGMT-23103) - Found and fixed phase regression bug** — phase was dropping to `Starting` on re-provision; mitigated by preserving `Running` phase when instance was already provisioned.
- **[MGMT-23103](https://issues.redhat.com/browse/MGMT-23103) - Found and fixed re-provision timeout bug** — playbook timed out when `runStrategy=Halted` because VM never becomes ready. Fix: skip wait step for Halted VMs (PR: osac-templates#8).
- **[MGMT-23103](https://issues.redhat.com/browse/MGMT-23103) - Removed VMI-level checks from E2E test** — VM-level printableStatus is sufficient; VMI checks were KubeVirt internals.
- **[MGMT-23103](https://issues.redhat.com/browse/MGMT-23103) - Re-vendored osac.templates in osac-aap** (PR merged: osac-aap#205). EE image rebuilt.
- **[MGMT-23103](https://issues.redhat.com/browse/MGMT-23103) - Found and fixed deprovision spec validation error** — finalizer role's server-side apply was rejected by immutability webhook; switched to JSON merge patch (PR: osac-aap#206).
- **[MGMT-23103](https://issues.redhat.com/browse/MGMT-23103) - Found and fixed tenant-not-ready requeue bug** — `handleUpdate()` was returning "done" instead of requeueing when tenant not ready.
- **[MGMT-23103](https://issues.redhat.com/browse/MGMT-23103) - Found and fixed tenant GC race condition** — new ComputeInstance would update a terminating shared tenant, causing stuck provisioning. Fix: detect terminating tenant, clear reference, and requeue.
- **[MGMT-23103](https://issues.redhat.com/browse/MGMT-23103) - Added `Updating` phase to ComputeInstance API** across three repos:
  - fulfillment-api: added `COMPUTE_INSTANCE_STATE_UPDATING = 5` to proto enum (PR: fulfillment-api#61)
  - fulfillment-service: added state to private API proto (PR: fulfillment-service#306, all 17 test suites pass)
  - osac-operator: added `Updating` phase with state machine refactor and comprehensive tests (PR: osac-operator#114, all 110 tests pass)
