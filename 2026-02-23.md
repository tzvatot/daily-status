# Daily Status - 2026-02-23

## Accomplishments

- **[MGMT-23031](https://issues.redhat.com/browse/MGMT-23031) – Changed default provisioning provider from EDA to AAP** in osac-operator (`cmd/main.go` and config sample updated). All tests pass, linter clean.
- **[MGMT-23103](https://issues.redhat.com/browse/MGMT-23103) – Fixed re-provision logic in operator**: added `needsProvisionJob()` to detect spec changes (e.g. runStrategy) and trigger re-provisioning; previously the operator only provisioned once.
- **[MGMT-23103](https://issues.redhat.com/browse/MGMT-23103) – Fixed phase regression bug**: phase was incorrectly dropping to `Starting` on re-provision of a Running ComputeInstance; mitigated by preserving `Running` phase when instance was already successfully provisioned.
- **[MGMT-23103](https://issues.redhat.com/browse/MGMT-23103) – Fixed re-provision timeout when runStrategy=Halted**: playbook waited for VM Ready=True on a Halted VM, causing a 600s timeout. Fix skips the readiness wait when `vm_run_strategy == "Halted"` (osac-templates PR #8 merged).
- **[MGMT-23103](https://issues.redhat.com/browse/MGMT-23103) – Fixed E2E test**: updated from `spec.running` to `spec.runStrategy` assertions, removed unnecessary VMI-level checks.
- **[MGMT-23103](https://issues.redhat.com/browse/MGMT-23103) – Re-vendored osac.templates in osac-aap** (PR #205 merged), rebuilt EE image, deployed updated operator to vmaas-dev.
