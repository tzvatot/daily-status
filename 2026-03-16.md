# Daily Status - 2026-03-16

## Accomplishments
- Agreed with Adrien on the approach for fixing infinite retry behavior in cluster provisioning: use a config version tracking mechanism combined with exponential backoff. When a job fails, the system will distinguish between failures due to spec changes (retry immediately) vs unchanged failures (retry with exponential backoff, no halt).
- Split the implementation into three incremental PRs to keep changes focused:
  - [osac-operator#110](https://github.com/osac-project/osac-operator/pull/110): Add config version tracking to job status and populate it when triggering provision jobs in ClusterOrder and HostPool controllers.
  - A follow-up PR to extend config version tracking to the ComputeInstance controller.
  - A second follow-up PR to implement the actual retry mechanism with exponential backoff and enable the pending integration tests.
